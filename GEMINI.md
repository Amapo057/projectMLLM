# 📖 Project Youkai (요카이) - AI Assistant Architecture & Guidelines

## 1. 프로젝트 개요 (Project Overview)
본 프로젝트는 로컬 PC 및 스팀덱(SteamOS/Linux) 환경에서 자비스(Jarvis)처럼 작동하는 상시 대기(Always-on)하는 **음성 기반 데스크탑 AI 비서 파이프라인**을 구축하는 것을 목표로 한다. 
호출어는 **"요카이(Youkai)"**이며, 시스템 리소스(배터리, CPU) 소모를 최소화하는 시스템을 만들어야 한다.

## 2. 코어 파이프라인 아키텍처 (Core Pipeline)
전체 시스템은 동기/비동기 혼합 파이프라인으로 구성되며, 마이크 점유율 충돌(Locking)을 방지하기 위해 각 단계 완료 시 자원을 완벽히 해제(`terminate`)하는 것을 원칙으로 한다.
화면 구현은 가장 마지막 목표이니, 아래의 지침에서 Screen, 화면관련 지침은 현재 무시한다.

0. **Python**
    * **버전:** `python 3.11.15` 사용
1. **Wake Word (호출어 감지)**
   * **기술:** `open wake word 0.6.0`
   * **전략:** `open wake word`의 colab 자체 학습으로 생성한 yo_kah_ee 호출어 사용
   * **장점:** 100% 오프라인 영구 무료, 라이선스/네트워크 제약 없음, CPU/배터리 점유율 극소화.
2. **STT (사용자 음성 인식)**
   * **기술:** STT Service (마이크에서 사용자 명령 텍스트화)
3. **Context Routing (외부 데이터 수집)**
   * **기술:** `requests 2.33.1`, REST API 연동
   * **구현:** `WeatherService` (Open-Meteo API). 어제와 오늘의 기온/날씨 데이터를 배열(`past_days=1`)로 가져와 제미나이의 프롬프트 컨텍스트에 주입.
4. **LLM (응답 생성)**
   * **기술:** Google `Gemini API` (`gemini-3.1-flash-preview` 등 빠른 모델 권장)
   * **역할:** System Instruction에 부여된 페르소나를 기반으로, 사용자 프롬프트와 Context 데이터를 융합하여 '비서다운' 자연스러운 브리핑 텍스트 생성.
5. **TTS (음성 출력)**
   * **기술:** `edge-tts 7.2.8` + `pygame 2.6.0`
   * **구현:** 비동기(Async)로 MP3 파일을 생성한 뒤, `pygame`으로 동기적 재생. 재생 완료 후 즉시 파일 점유(Lock) 해제. (Voice: `ko-KR-SunHiNeural` 등)
6. **Screen**
    * **기술:** `PyQt6 6.11.0`
    * **구현:** 평시엔 전력 소모를 줄이기위해 완전한 검은색을 화면에 띄우고, 호출어 작동시 화면 중앙에 아이콘을 띄워 시각적으로 상태를 볼 수 있도록 표시.

## 3. 앱 동작 루틴:
1. 앱 시작 → 대기 모드 (PyQt6 윈도우 검은 화면 유지 + 백그라운드 호출어 상시 감지)
2. 호출어 감지 → 대기 해제 → STT 자동 시작 및 중앙 Orb UI 활성화 (지정된 시간 무응답 시 대기 모드 자동 복귀)
3. 음성 인식(STT) → 텍스트 파싱 및 키워드 감지 → 외부 API(날씨, 버스 등) 호출 및 정보 수집 → Gemini 프롬프트 삽입 → 응답 생성
4. 생성된 응답 TTS 출력 → 완료 후 다시 대기 모드 및 화면 갱신 중지 자동 진입

## 4. 개발 및 설계 철학 (Design Principles)
이 프로젝트의 코드를 작성하거나 수정할 때 Gemini는 반드시 다음 원칙을 준수한다.

* **최적화 우선 (Resource Efficiency):** 스팀덱 환경을 고려하여 백그라운드 상시 대기 모듈은 가장 가벼운 방식을 채택한다.
* **오프라인 지향 (Local First):** 호출어 감지 등 치명적인 대기 전력 소모 구간은 클라우드나 기업용 API(Picovoice 등)를 피하고 로컬에서 해결한다.
* **버전 충돌 회피:** 파이썬 3.12 이상의 환경 및 SteamOS(Arch Linux) 환경에서 C 언어 의존성(Numpy 등) 충돌이 발생하지 않는 검증된 패키지만 사용한다.
* **프롬프트 엔지니어링 통제:** LLM에게 직접 연산을 시키지 않고, 코드 단에서 외부 API를 통해 데이터를 수집 및 비교(`어제 기온 X, 오늘 기온 Y`)하여 프롬프트의 Context로 밀어 넣는다.

## 5. 향후 추가 과제 (Future Milestones)
* **효과음(SFX) 추가:** 호출어 인식 직후 STT 대기열로 넘어갈 때 "네?" 하는 피드백 사운드 추가 (사용자 경험 향상).
* **아두이노와 블루투스 연결:** 아두이노와 블루투스로 연결해 패킷을 만들어 통신해 불을 끄는 동작 추가(light_switch코드 참조).
